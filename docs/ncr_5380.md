# NCR 5380 / 53C80 SCSI Host Controller

This document describes the **NCR 5380** (and pin-compatible 53C80 / DP5380)
host-adapter chip in enough detail to emulate it correctly for every
Macintosh that uses it.  Machine-specific wiring — memory map, interrupt
routing, pseudo-DMA regions — is covered separately in
[scsi.md](scsi.md).  This file is chip-only.

The 5380 is an asynchronous, 8-bit, single-ended SCSI-1 controller.  It
manages the low-level REQ/ACK handshake and arbitration but **performs no
memory access itself**; data movement is the responsibility of the host
CPU or an external DMA engine.  That design is what makes "pseudo-DMA"
possible on Apple's 68k machines (see [scsi.md §2](scsi.md)).

---

## 1. Architecture overview

* **Eight register-selectable addresses.**  A2–A0 select a register;
  read vs. write selects which of the two possible registers at that
  address is accessed (CSR/ODR at 0, CBSR/SER at 4, etc.).
* **No fixed pin for DACK** — instead, two groups of "Start DMA …"
  register-write triggers and a separate DRQ pin.  A pseudo-DMA
  implementation maps accesses at special host addresses onto the
  appropriate DMA register.
* **Open-collector / wired-OR** SCSI bus drivers; the chip does not
  terminate the bus.
* **Asynchronous REQ/ACK only.**  Synchronous-offset negotiation
  (SDTR) must be rejected or ignored by the initiator firmware.
* **Three modes of operation** determined by register writes:

  1. **Programmed I/O** — host drives every REQ/ACK by toggling `ICR`.
  2. **DMA mode** — chip drives REQ/ACK; host supplies bytes on DRQ /
     DACK.  On Apple machines, "DACK" is synthesised by memory-space
     decoding of pseudo-DMA address regions (see [scsi.md §3](scsi.md)).
  3. **Target mode** (via `MR.TargetMode=1`) — the chip responds to
     selection from another initiator.  Macintosh host firmware rarely
     uses target mode, but the chip's loopback / diagnostic test card
     exercises it.

---

## 2. Register map

The 5380 exposes **eight logical registers**, selected by A2–A0, with
independent read and write sides at four of the eight addresses.

| A2 A1 A0 | Read side                            | Write side                                |
| -------- | ------------------------------------ | ----------------------------------------- |
| 000      | Current SCSI Data (**CDR**)          | Output Data Register (**ODR**)            |
| 001      | Initiator Command Register (**ICR**) | Initiator Command Register (**ICR**)      |
| 010      | Mode Register (**MR**)               | Mode Register (**MR**)                    |
| 011      | Target Command Register (**TCR**)    | Target Command Register (**TCR**)         |
| 100      | Current SCSI Bus Status (**CSR**)    | Select Enable Register (**SER**)          |
| 101      | Bus and Status Register (**BSR**)    | Start DMA Send                            |
| 110      | Input Data Register (**IDR**)        | Start DMA Target Receive                  |
| 111      | Reset Parity / Interrupt             | Start DMA Initiator Receive               |

"CDR" and "IDR" are normally identical on read — both return the DB[7:0]
latch after the most-recent ACK handshake.  Writing ODR drives the data
bus when `ICR.AssertDataBus` is set or the chip is in target mode.

Register codes used in this emulator's source tree:

* `CDR=0`, `ODR=0`, `ICR=1`, `MR=2`, `TCR=3`, `CSR=4`, `SER=4`,
  `BSR=5`, `DMA=5` (Start DMA Send), `IDR=6`, `TDMA=6`
  (Start DMA Target Receive), `IDMA=7` (Start DMA Initiator Receive),
  `RESET=7` (Reset Parity/Interrupt).

See [src/core/peripherals/scsi_internal.h](../src/core/peripherals/scsi_internal.h).

## 3. Registers in detail

### 3.1 CDR / ODR — data path (address 0)

* **CDR (read)**.  Transparent view of the SCSI DB[7:0] signals, gated
  and re-latched on every incoming REQ when the chip is initiating
  (or on outgoing ACK when targeting).  A CDR read at a pseudo-DMA
  address *with* `MR.DMA=1` and DRQ asserted transfers one byte and
  issues a matching ACK.
* **ODR (write)**.  Drives DB[7:0] when either
  `ICR.AssertDataBus=1` or `MR.TargetMode=1`, or as the arbitration
  ID bit pattern.  ODR writes at a pseudo-DMA address (`Start DMA
  Send`-aliased) with DMA mode and DRQ asserted consume one byte
  and advance REQ/ACK.
* **IDR (read, address 6)** is a synonym of CDR on the 5380.

### 3.2 ICR — Initiator Command Register (address 1)

Write bits drive initiator-role control lines; read bits report latch
state.

| Bit | Write meaning            | Read meaning             |
| --- | ------------------------ | ------------------------ |
| 7   | ASSERT RST               | ASSERT RST (latched)     |
| 6   | TEST MODE                | ARBITRATION IN PROGRESS  |
| 5   | DIFF ENBL *(5381 only)*  | LOST ARBITRATION         |
| 4   | ASSERT ACK               | ACK                      |
| 3   | ASSERT BSY               | BSY                      |
| 2   | ASSERT SEL               | SEL                      |
| 1   | ASSERT ATN               | ATN                      |
| 0   | ASSERT DATA BUS          | DB asserted              |

Side effects:

* Setting **ASSERT RST** clears all other registers except this bit
  itself and the interrupt latch, and raises the bus reset IRQ source.
* **ACK / ATN edges** are the mechanism by which the initiator drives
  single-byte handshakes during PIO.  The emulator uses the **falling
  edge of ACK** as the trigger to advance bus phase (command → data,
  data_in → status on exhaustion, status → message_in, message_in →
  bus_free).  See `write_icr` / `bits_cleared & ICR_ACK` in
  [scsi.c](../src/core/peripherals/scsi.c).

### 3.3 MR — Mode Register (address 2)

Controls chip-wide behaviour.

| Bit | Name           | Effect                                              |
| --- | -------------- | --------------------------------------------------- |
| 7   | BLOCK DMA      | DMA runs in block mode (assert DACK for full burst) |
| 6   | TARGET MODE    | Chip responds to selection as a target              |
| 5   | ENBL PARITY CK | Check incoming parity                               |
| 4   | ENBL PARITY IR | Raise IRQ on parity error                           |
| 3   | ENBL EOP IR    | Raise IRQ on end-of-DMA                             |
| 2   | MONITOR BUSY   | Raise IRQ if BSY deasserts during DMA               |
| 1   | **DMA MODE**   | Enable automatic REQ/ACK handshake via DRQ/DACK     |
| 0   | ARBITRATE      | Begin arbitration using ODR as ID                   |

Two bits drive all the interesting host-software behaviour:

* **DMA MODE (MR.1)** turns the chip into a "feed me bytes" state
  machine.  With this bit set and the bus in an information-transfer
  phase that *matches* TCR, the chip asserts **DRQ** and waits for the
  host to read CDR (or write ODR) via the pseudo-DMA alias.  Each such
  access counts as an implicit ACK.
* **ARBITRATE (MR.0)** latches ODR as the initiator ID, raises
  `ICR.ArbitrationInProgress`, and advances the internal arbitration
  state machine.  Apple SCSI Managers on the Mac rarely set this —
  they use **non-arbitrated selection** (see [scsi.md §4.1](scsi.md)).
  The emulator's arbitration implementation always "wins" and
  captures ODR's host ID.

Setting MR_DMA while the bus phase does **not** match TCR does
**not** assert DRQ — instead the chip raises the **phase-mismatch
IRQ** (see §3.6).  This is the mechanism that drives Mac OS's surprise
CHECK_CONDITION recovery path — the emulator models this precisely by
gating `BSR_DR` assertion on `scsi_phase_match()` in `write_mr`.

### 3.4 TCR — Target Command Register (address 3)

Bits 2:0 encode the phase that either (a) the chip drives when in
target mode or (b) the initiator expects when DMA is active.  The chip
uses bits 2:0 of TCR together with the current bus signals to compute
**phase match** (see §3.5).

| Bit | Name        |
| --- | ----------- |
| 0   | Assert I/O  |
| 1   | Assert C/D  |
| 2   | Assert MSG  |

The standard SCSI phase encodings (what the initiator sets TCR to
*before* enabling DMA):

| Phase       | MSG | C/D | I/O | TCR value |
| ----------- | --- | --- | --- | --------- |
| DATA OUT    | 0   | 0   | 0   | `0x00`    |
| DATA IN     | 0   | 0   | 1   | `0x01`    |
| COMMAND     | 0   | 1   | 0   | `0x02`    |
| STATUS      | 0   | 1   | 1   | `0x03`    |
| MESSAGE OUT | 1   | 1   | 0   | `0x06`    |
| MESSAGE IN  | 1   | 1   | 1   | `0x07`    |

> Ignore reserved TCR bit-7..3 on read; they're unused on the 5380.

### 3.5 CSR — Current SCSI Bus Status Register (read at address 4)

Untransformed snapshot of the live SCSI control signals.

| Bit | Signal |
| --- | ------ |
| 7   | RST    |
| 6   | BSY    |
| 5   | REQ    |
| 4   | MSG    |
| 3   | C/D    |
| 2   | I/O    |
| 1   | SEL    |
| 0   | DBP    |

Host software uses CSR to poll for phase transitions during PIO and
to sample BSY during arbitration.  Bits 4..2 give the current SCSI
phase; compared against TCR[2..0], they drive the "phase match" bit
in BSR.

### 3.6 BSR — Bus and Status Register (read at address 5)

| Bit | Name                 | Meaning                                                        |
| --- | -------------------- | -------------------------------------------------------------- |
| 7   | END OF DMA (EDMA)    | DMA terminated (EOP asserted, or BSY lost in DMA)              |
| 6   | **DMA REQUEST (DR)** | Chip is requesting a DMA transfer (DRQ pin asserted)           |
| 5   | PARITY ERROR         | Parity mismatch latched (cleared by Reset Parity/Interrupt)    |
| 4   | **INT REQUEST**      | IRQ pin asserted (cleared by Reset Parity/Interrupt)           |
| 3   | **PHASE MATCH (PM)** | `CSR[4..2] == TCR[2..0]`                                       |
| 2   | BUSY ERROR           | BSY was deasserted outside the expected phase                  |
| 1   | ATN                  | ATN asserted on the bus                                        |
| 0   | ACK                  | ACK asserted on the bus                                        |

DR, PM, and INT are the three bits polled by every host driver:

* **DR** goes high when DMA mode is active *and* phase matches TCR
  *and* the target has REQ asserted with a byte ready.
* **PM** is a pure combinational derivation; the emulator recomputes
  it on every BSR read (`scsi_phase_match()`).
* **INT** is the latched interrupt flag.  Sources include phase
  mismatch during DMA, end-of-DMA, bus reset, parity error, and
  (on 53C80) parity error on command phase.  Reading
  Reset Parity/Interrupt clears bits 5, 4, and 2.

### 3.7 SER — Select Enable Register (write at address 4)

Writing a byte with bit *n* set tells the chip to raise the **Select**
interrupt when a foreign initiator selects ID *n*.  Macintosh host
firmware sets SER to `1 << 7` (the host's own ID) to detect
reselection from targets that support disconnect / reconnect.

### 3.8 "Start DMA …" triggers (write at addresses 5, 6, 7)

These three register writes arm the DMA engine for a specific
direction:

* **Start DMA Send** (addr 5, write) — initiator → target transfer.
* **Start DMA Target Receive** (addr 6, write) — target-mode transfer
  from initiator.
* **Start DMA Initiator Receive** (addr 7, write) — initiator receive
  from target.

Per datasheet §6.8.1, these are **mandatory** to begin DMA: until the
host writes the appropriate Start DMA register, ODR-alias writes are
absorbed by the chip's data register but do **not** propagate to the
SCSI bus.  More subtly, even after Start DMA Send the chip does not
transmit a byte until the target asserts REQ; ODR-alias writes that
arrive before the first REQ overwrite each other, and only the
most-recent value reaches the bus.

The Apple pseudo-DMA driver paths typically just set `MR.DMA=1`,
program TCR for the expected phase, and immediately begin moving
data through the pseudo-DMA address aliases.  However, the **A/UX
3.0.1 SE/30 SCSI driver** exploits the "previous ODR overwritten
before first REQ" semantics: it issues an unhandshaken `CLR.B
([$5B20E,])` at PC `$1004B888` (and three other call sites) to the
SE/30 BLIND pseudo-DMA region (`$50012000`) immediately before its
byte loop.  Real hardware latches that `$00` into ODR but doesn't
transmit it, because the byte loop's first real data byte overwrites
the primer in ODR before REQ is asserted.

The emulator models this in two layers:

1. `dma_write_armed` (cleared on `MR.DMA` rising edge, set on Start
   DMA Send write) silently drops ODR-alias writes that arrive
   before Start DMA Send.  This handles the "between MR.DMA enable
   and Start DMA Send" window.
2. `primer_held` (in `data_out` only) holds the first ODR-alias
   write of the phase in stasis and decides on the second write:
   if the held byte is `$00` and the second write comes from a
   different PC than the first, the held byte was a kernel primer
   and is discarded; otherwise it is real data and is pushed
   ahead of the second.  PC-discrimination distinguishes A/UX's
   distinct-PC primer pattern (CLR.B + byte-loop) from MacOS's
   tight-loop data writes (all bytes at one PC).

`MR.DMA=0` keeps the legacy PIO-style path: ODR-alias writes during
`command`/`data_out` are consumed unconditionally because the
host's ICR.ACK handshake provides the per-byte gating.  See
[notes/60-aux3-volname-root-cause.md](../local/gs-docs/notes/60-aux3-volname-root-cause.md)
for the investigation history.

### 3.9 Reset Parity / Interrupt (read at address 7)

Reading this register clears the latched parity-error, interrupt, and
busy-error bits in BSR.  It does **not** affect the DMA-request or
phase-match bits; nor does it clear the `ICR.ASSERT RST` latch.

## 4. Bus phase tracking

On real hardware, bus phase is determined entirely by the target
(MSG/C/D/I/O are target-driven).  The chip exposes it via CSR[4..2]
and derives PM by comparing against TCR.  The emulator models this
with an explicit `scsi_phase_t` enum and updates CSR whenever the
phase changes:

* `scsi_bus_free` — CSR=0
* `scsi_arbitration` — CSR.BSY driven by the host
* `scsi_selection` — CSR.BSY/SEL
* `scsi_command` — CSR.CD|REQ|BSY
* `scsi_data_in` — CSR.IO|REQ|BSY
* `scsi_data_out` — CSR.REQ|BSY
* `scsi_status` — CSR.IO|CD|REQ|BSY
* `scsi_message_in` — CSR.IO|CD|MSG|REQ|BSY
* `scsi_message_out` — CSR.MSG|CD|REQ|BSY (target-driven after ATN)

Phase transitions happen only on well-defined triggers (REQ/ACK
completion, end-of-buffer, target-initiated phase change).  See
`phase_*()` helpers in [scsi.c](../src/core/peripherals/scsi.c).

## 5. Arbitration and selection

### 5.1 Arbitrated selection

1. Host writes its own ID to ODR (e.g. `1 << 7`).
2. Host sets `MR.ARBITRATE=1`.  Chip asserts `ICR.AIP`.
3. After bus-free + arbitration-delay timers elapse, chip reports
   whether it won (`ICR.LA=0` and `ICR.AIP=1`).
4. Host transitions to selection by asserting `ICR.SEL` and writing
   the *target-ID OR host-ID* bit pattern to ODR, then deasserting
   BSY via `ICR.BSY=0`.
5. Target responds by asserting BSY; chip advances to COMMAND phase.

### 5.2 Non-arbitrated selection

All classic Macintosh firmware (Plus ROM, SE/30 ROM, A/UX) uses
**non-arbitrated** selection because the host is the only initiator
on the bus:

1. Host writes target-ID bit to ODR, asserts `ICR.SEL`, `ICR.DATA`.
2. Host clears BSY; the chip jumps directly to the SELECTION phase
   without running the arbitration state machine.
3. Target responds by driving BSY.

The emulator captures this in `write_icr`'s "`BSY cleared during
selection`" branch.  When MR.ARBITRATE was never set, it assumes the
host ID is 7 (the canonical Mac host ID).

## 6. Interrupt sources

The 5380 raises **IRQ** (pin) whenever one of the following latched
events occurs, provided the corresponding MR enable bit is set:

* **Phase mismatch during DMA** — `MR.DMA=1` and `CSR[4..2] != TCR[2..0]`.
* **End of DMA (EOP)** — external EOP pin asserted or BSY dropped
  during DMA.
* **Bus reset** — SCSI RST edge detected.
* **Parity error** — if `MR.ENBL PARITY IR=1`.
* **Select** — foreign initiator selected the host's ID (SER match).

IRQ stays asserted until the **Reset Parity/Interrupt** register is
read.  The emulator drives a single `scsi_update_irq()` helper that
recomputes the aggregate condition and toggles the IRQ pin into the
host's interrupt glue — VIA2 CB2 on SE/30, unwired on Plus.  Chip-level
emulation does **not** decide whether IRQ leads to a CPU interrupt;
that is the host's job (see [scsi.md §3](scsi.md)).

---

## 7. Timing model

The 5380 is fully asynchronous; timing is dominated by REQ/ACK
handshake delays (typically 100–200 ns per handshake) and a few
programmed delays:

* **Bus-free settle**: ≥ 800 ns
* **Arbitration delay**: ≥ 2.4 µs
* **Selection timeout**: 250 ms per SCSI-1

For emulator fidelity the most important constraint is that
`phase_data_in` and `phase_data_out` transitions **do not fire
`scsi_update_irq`** during the first REQ assertion (deferring the
phase-mismatch IRQ until the host notices on its own).  See the
comment at `phase_data_in()` in
[scsi.c](../src/core/peripherals/scsi.c).  The Apple SCSI Managers
run pseudo-DMA at low IPL and would otherwise take a spurious nested
interrupt.

---

## 8. Read-to-clear side effects

Several reads have latch-clearing side effects that must be modelled:

* **Reset Parity / Interrupt** (addr 7, R) — clears BSR bits 5, 4, 2.
* **CDR/IDR** (addr 0, R) with `MR.DMA=1` and PM satisfied — consumes
  one byte from the REQ/ACK handshake and advances the internal
  buffer.  Without DMA mode, CDR reads are side-effect-free.
* **CSR** — pure snapshot, no side effects.

---

## 9. Reset behaviour

Two paths reset the chip:

* **Software RST** (`ICR.ASSERT RST=1`).  Clears all registers except
  `ICR.ASSERT RST` itself and the interrupt latch; raises the
  interrupt with the RST latch bit set.  Chip stays in reset while
  the bit is high; on its falling edge, ICR is cleared too.
* **Power-on reset**.  Same final state, but no latched interrupt.

The emulator's `scsi_reset()` mirrors this: it clears all POD state
(registers, buffer, EOP flag, CDR pipeline), re-asserts `BSR.INT`,
and fires `scsi_update_irq` + `scsi_update_drq` so the host's glue
sees the edges.
